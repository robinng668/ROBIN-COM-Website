## 分级汇报机制

### 等级划分

| 等级 | 权限 | 汇报条件 |
|------|------|----------|
| **L0** | 执行权 | 日常任务，直接执行 |
| **L1** | 建议权 | 需要确认的决策 |
| **L2** | 否决权 | 重大事项需审批 |
| **L3** | 决策权 | 战略方向（卡特专属） |

### Agent权限

| Agent | 等级 | 自主范围 |
|-------|------|----------|
| 发哥 | L1 | 网站/VNPY/报告执行 |
| 星星 | L1 | 投研/回测执行 |
| 巴菲 | L1 | 价值分析建议 |

## 汇报流程

```
┌─────────────────────────────────────────────────────────────┐
│                      任务执行流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 目标设定 (L3 卡特)                                      │
│       ↓                                                     │
│  2. Agent自主思考 → 决定优先级 → 执行                        │
│       ↓                                                     │
│  3. 执行中遇到问题？                                        │
│       ├─ 能解决 → 自己解决 → 继续执行                        │
│       └─ 需要确认 → 发起请示 → 卡特审批 → 继续执行            │
│       ↓                                                     │
│  4. 完成后 → 分级汇报                                       │
│       ├─ L0 结果 → 记录存档                                  │
│       └─ L1+ 结果 → 卡特审核                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 汇报类型

### 1. 日常汇报（L0）
- 任务完成报告
- 数据更新完成
- 正常运行确认

**格式：**
```
[完成] 任务名称 - 结果摘要
```

### 2. 建议汇报（L1）
- 需要确认的决策
- 方案选择
- 资源申请

**格式：**
```
[建议] 事项描述
方案A: ...
方案B: ...
建议: 方案X（理由）
请确认
```

### 3. 紧急汇报（L2）
- 风险预警
- 异常情况
- 重大发现

**格式：**
```
[紧急] 事项描述
影响: ...
建议: ...
请指示
```

## 执行规则

### Agent自主决策范围
- 任务执行顺序
- 技术方案选择
- 代码实现细节
- 数据获取方式

### 需要卡特确认的事项
- 重大决策（资金使用 > 1万）
- 战略方向调整
- 新增任务范围
- 外部合作
- 对外发布

### 禁止事项（必须汇报）
- 发现风险
- 策略失效
- 系统异常
- 外部威胁
