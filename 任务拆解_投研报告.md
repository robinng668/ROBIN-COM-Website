## 三、报告结构（V9.0规范）

### 3.1 页面分布

| 页码 | 内容 |
|------|------|
| P1 | 封面：核心主题+重点主线 |
| P2 | 1.1 市场概览+点评 |
| P3 | 1.2 驱动因素+标的 + 1.3 风险+即将事项 |
| P4-Pn | 各行业分析（五维+个股+小结） |
| 末页 | 投资建议+风险提示 |

### 3.2 数据列要求

| 表格 | 必须列 |
|------|--------|
| 1.1市场指标 | 指标+数值+**点评** |
| 1.2驱动因素 | 主题+描述+**标的推荐** |
| 1.3风险提示 | 风险+量化+**即将事项** |
| 个股表格 | 名称+逻辑+数据+**关联事项**+**万德2026E** |

## 四、技术方案

### 4.1 数据源

| 数据类型 | 来源 | 接口 |
|----------|------|------|
| 实时行情 | Futu API | get_stock_quote |
| K线数据 | Futu API | get_kline |
| 宏观数据 | Baostock | query_macro |
| 研报摘要 | Web采集 | Tavily Search |
| 一致预期 | 模拟/估算 | - |

### 4.2 生成流程

```
T+0 17:00
    ↓
1. 获取实时行情（收盘价）
    ↓
2. 搜索今日热点资讯（Tavily）
    ↓
3. 提炼3个核心主题
    ↓
4. 生成1.1市场概览
    ↓
5. 生成1.2驱动因素（含标的）
    ↓
6. 生成1.3风险提示（含即将事项）
    ↓
7. 按景气度排序行业
    ↓
8. 生成各行业分析
    ↓
9. Markdown → HTML → PDF
    ↓
10. 推送到飞书群
```

## 五、自动化配置

### 5.1 Cron任务

```bash
# 每天 17:30 自动生成
30 17 * * * python3 /mnt/d/Aproject/Xfile/workbench/scripts/generate_pro_report.py

# 每天 18:00 飞书发送
0 18 * * * python3 /mnt/d/Aproject/Xfile/workbench/scripts/send_report.py
```

### 5.2 输出文件

```
/mnt/d/Aproject/Xfile/workbench/analysis/
├── 投研报告_V9_YYYYMMDD.pdf    # 最终PDF
├── 投研报告_V9_YYYYMMDD.html    # HTML备份
└── 投研报告_V9_YYYYMMDD.md      # Markdown源
```

## 六、产出标准

### 6.1 内容标准

- [ ] 至少6个重点行业
- [ ] 每行业至少5只个股
- [ ] 至少10只推荐标的
- [ ] 每只标的有万德2026E PE（或估算）
- [ ] 每行业有行业小结

### 6.2 格式标准

- [ ] 封面完整（标题/日期/主题/主线）
- [ ] 1.1表格有点评列
- [ ] 1.2有推荐标的+理由
- [ ] 1.3有即将发生事项
- [ ] 行业按景气度排列

### 6.3 时间标准

- [ ] 17:30启动生成
- [ ] 18:00前完成PDF
- [ ] 18:05前推送完成
